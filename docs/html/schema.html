Library schema

Single SQLite database at ~/Library/Application Support/Dub/library.sqlite. The schema is part of Dub's public API; we do not break the contract without a documented migration path and a schema_version bump. The PRD goal is that a user who wants to leave Dub takes their data with them; no encryption, no proprietary formats, every decision is queryable.

Schema v2 (M11d.4) · WAL + foreign_keys ON + 256 MB mmap · 15+ tables

Entity relationship overview

Click any table in the diagram to jump to its column reference. The canonical track identity is tracks.id (a UUID TEXT); per-source opinions are preserved verbatim by source via track_metadata_source. File identity is by (volume_uuid, relative_path) so drive moves don't break.

tracks canonical identity

The canonical track identity row. id is a 36-character lowercase RFC 4122 UUID stored as TEXT. duplicate_link_track_id is set when version-aware dedupe detects a potential duplicate (PRD §8.1); the link is bidirectional. explicit_merge_target_id is set by user-driven merge (v1.x) and lets the merged-from row survive as a tombstone so re-import doesn't resurrect it.

fingerprints Chromaprint algorithm 2 · pure-Rust

Keyed against the canonical recording, not the file. Two track_files pointing at the same recording from different drives share one fingerprint row via tracks.fingerprint_id. chromaprint_blob is raw little-endian u32 items, no header, length = bytes.len() / 4. Generated by rusty-chromaprint with preset_test1 (11025 Hz · 4096-frame FFT · 2/3 overlap).

track_files files on disk schema v2 missing-files

One canonical track may have many files (encodings, backups, touring SSD + studio drive). Resolution order on load (PRD §8.2): volume UUID → last-known mount point + relative path → basename + fingerprint search → prompt the user. The background scanner sets is_missing; a track is reported as missing iff every track_files row for it has is_missing = 1.

track_metadata_source per-source verbatim opinion

The single most important table in the schema: every source's verbatim opinion is preserved. We never collapse Serato's "Dilla, J" and rekordbox's "J Dilla" into one row at import time. track_metadata_fts FTS5 virtual table

Substring match with AND across whitespace-separated tokens per PRD §8.5.4. Operators (bpm:90-100, key:Am) are parsed out of the query string before the FTS5 call and applied as separate SQL predicates. unicode61 remove_diacritics 2 so "Beyoncé" matches "Beyonce" and vice versa.

schema_version single-row · migration anchor

Single-row table; one INSERT at initial schema creation, one UPDATE per migration. The migration runner is a sorted list of (target_version, sql_script) pairs; each script is idempotent against partial application so a crash mid-migration leaves a consistent state. A v1.0 binary opening a v1.x-written database detects schema_version > SCHEMA_VERSION and refuses to write (LibraryError::SchemaTooNew).

ColumnTypeNotes

PRAGMAs and conventions

Per-connection PRAGMAs

PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
PRAGMA foreign_keys = ON;
PRAGMA temp_store = MEMORY;
PRAGMA mmap_size = 268435456;

WAL is critical: the importer worker thread writes while the browser read query streams rows. Without WAL the writer blocks all readers for the duration of an import. foreign_keys = ON is non-negotiable; SQLite's default is OFF for historical reasons.
