Roadmap · M0 through v1.1

Roadmap

Where Dub has been and where it's going. Shipped milestones link into anchored sections of SHIPPED.md; planned milestones link into PRD.md §12. Approximate aggregate to v1.0 is ~20–26 weeks of focused work; the Beta → Stable gap is intentionally variable since stable ships when the SLOs are met, not on a calendar.

Staged rigor (PRD §2.2.0)

Reliability gates apply progressively. Pre-alpha runs with all the framework operational (TDD, CI, RT-safety, fuzzing) but no gig-hour gates; alpha adds the manual rig checklist; stable adds the full §2.2.6 SLOs.

Phase A · Pre-Alpha (M0 → M17) TDD discipline · RT-safety enforcement · parser fuzzing · CI green to merge · snapshot tests · branch protection.
"Release" = author dogfoods latest main daily.
Phase B · Alpha (M18) + nightly soak harness · + manual rig checklist · + 24 h hotfix discipline for crashes reported by 3–5 trusted DJs.
Phase C · Beta & Stable (M19, M20+) + full §2.2.6 SLOs · 100 cumulative beta-gig-hours zero-crash · zero fuzz crashes in last 7 days · no benchmark regressions.

PRD.md §12.2

Out of scope

These are listed explicitly so that "is it in scope?" never needs to be inferred from silence. Most are deferred to v2 (or later); a few are deferred to v1.x with a clear reason.

Deferred to v2 or later

  • Internal mixer / software EQ / crossfader / cue channel / pitch fader
  • Hot cues (entire system · no v1 lite version)
  • Recording (master out + per-deck)
  • Streaming services
  • Phase support
  • HID controllers
  • Stems / AI separation
  • Video / OBS
  • Windows port
  • Mac App Store distribution

Deferred to v1.x

  • Saved loop slots (v1 ships ephemeral loops only)
  • Sampler beyond 4 slots
  • Phase-Drift Trail numeric-only Preferences variant
  • Track Preparation Mode editing tools (beatgrid editor, hot-cue prep, gain UI)
  • Additive waveform layers (phrase landmarks, ghost waveform, vocal-presence overlay)
