Dub — timecode-vinyl DJ application for scratch DJs

Mac-first, Rust-cored, GPLv3. Built around one uncompromising commitment: best-in-class control-vinyl latency and feel on macOS, plus first-class support for playing real records through the software with FX, auto-BPM, and (eventually) automatic waveform recognition. The audience is the urban / sound-system / scratch DJ; reliability is the primary feature, not a feature.

Non-negotiable design principles

These are load-bearing. The PRD treats them as binding for every line of code. Anything that contradicts one of these needs explicit agreement before it lands.

No Mouse DJ Ever

Performance gestures (pitch, scratch, crossfade, EQ, gain, cue) never go through the mouse. They live on the turntable + external mixer + keyboard. The mouse is welcome for navigation, config, and the explicit transport / recovery affordances.

External mixer is the product

No software mixer / EQ / crossfader in v1 or v2. Dub requires a ≥4-in/4-out audio interface; the user's hardware mixer does the mixing.

Real records are first-class

Thru mode auto-detects when a real record is on the deck and switches transparently. FX work on real records the same way they work on files.

Reliability over features

A crash on stage is a career moment for a DJ. We accept ~20–30 % slower velocity to never ship a show-stopper. TDD is not optional and the audio thread is statically guarded.

Symmetric playback

Forward and backward playback are byte-for-byte symmetric. Manual rewinds are a first-class gesture. Whole tracks are decoded into RAM on load (no per-block disk streaming).

The audio thread is sacred

No alloc, no lock, no syscall, no logging, no I/O, no unwrap(), no dyn Trait heap allocation. Enforced at compile time via the RealtimeContext token and at runtime via assert_no_alloc.
