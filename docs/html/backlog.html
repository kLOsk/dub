UI backlog

Items found during the 2026-05-17 SwiftUI review that were not immediately fixed in-session. Triaged into three buckets ordered roughly by priority within each. Closed and stale items are kept at the bottom for archaeology; the kanban above only shows open work.

1 · Real bugs correctness or visible user pain
B-7low impact · wasteful
scanMissingFilesBatch has identical if / else branches
Both arms call library.markFileState with the same arguments. Behaviour is correct; it just writes the same row to SQLite twice the database hits otherwise would on first-import days.

Fix: collapse the branches, or split the FFI so the unchanged path is a cheap UPDATE … SET last_seen_at = ?.
swift MainView.swift ~L957
B-8selection
libraryTrackCount refresh clobbers selection
The onChange(of: model.libraryTrackCount) path calls refreshTracks() with preserveSelection: false. Intended trigger is "another window just imported tracks"; in practice it also fires during the user's own session and loses the highlighted row.

Fix: pass preserveSelection: true from this onChange. The preserve path already exists.
swiftLibraryView.swift
B-9scales to 100 k rows
Search field has no debounce
Every keystroke re-queries FTS5 synchronously and triggers a full list rebuild. Fast today, but a 500 ms-long word means six queries plus six rebuilds.

Fix: wrap the onChange in .task(id: searchText) with a 250 ms sleep. Cancel-on-new-input is automatic with that pattern.
swiftLibraryView.swift

Dev-facing placeholder text leaks into Casual Play

Fix: replace with user-facing copy that names the feature, not the milestone, or hide placeholders entirely in Casual-Play builds behind a debug flag.

Error toasts are too aggressive
surfaceError lights up a full-width banner for any FFI hiccup (transient unmounted volume, stale path lookup, short-track analysis failure). Users in the middle of a mix should not get attention-stealing surfaces.

Fix: split into user-actionable (banner) vs informational (status-strip glyph or quiet log line). Engine / library open / Preferences errors stay in the banner; analysis and resolve failures go to the log.

U-18 dup ID in MD
Sort semantics differ between FFI and client sort
FFI's list_tracks_sorted puts NULLs last both directions. Client-side KeyPathComparator over optional keys puts NULLs last for ASC and first for DESC, so clicking Artist twice gives different NULL placement than the initial open.

Fix: build a NULL-always-last comparator, or push the sort back through the FFI so there's one source of truth. FFI path is preferable once paging lands.

U-19 dup ID in MD
Idle-pane hint text gets truncated
Below ~840 px window width, the second line of Placeholders.idleHint truncates mid-word ("…drop a track to load i").

Fix: drop .lineLimit(2) in favour of .lineLimit(nil) + .minimumScaleFactor(0.8). Allowing the hint to wrap to three lines on narrow windows is better than truncated text.

3 · Code health no symptom yet · architectural debt
C-27
selectLibraryTrack calls trackPath twice
Selection path: selectLibraryTrack calls trackPath to set browserSelection; on Space (or drag), recordLibraryLoadIfApplicable calls trackPath again to verify the URL hasn't changed. Same SELECT executed twice within a few ms.

Fix: cache the resolved URL on selectedLibraryTrack (snapshot already published) and have recordLibraryLoadIfApplicable compare against the cached value. The unmounted-mid-action edge case still trips the engine error.
swift
C-28
analysisInFlightCount naming is misleading post-fix
Post-fix for B-3/B-4 the counter is "any analysis in flight, batch or not" but the doc comment still says "divides batch total for progress fraction". Right value for batch progress is now analysisBatchCompleted.

Fix: rename to analysisInFlight: Bool (we never use count > 1; analyses are serial). Trivial; lands with the next library-related PR.
swift
C-30
RT-thread audit doesn't cover the SwiftUI render path
The RT-thread auditor walks the engine's audio callback. The 30 Hz UI poll is RT-safe by construction (main actor) but the path between an audio-thread BPM update and a deck-header refresh routes through several FFI hops auditable only on the Rust side. Swift-side allocations during the poll are uncharacterised.

Fix: add a Swift-side MainActor profile pass that runs the 30 Hz poll for 60 s under Instruments' Allocations template. Not a hard "must be zero" gate; we want a baseline so regressions are visible.

C-24 · FileBrowserView.swift dead code · deleted

Deleted. No call sites after the M11d library superseded it (only its own #Preview + stale doc-comment references, which were rewritten). The XcodeGen sources: - path: Dub directory glob drops it on the next bootstrap.

C-25 · LibraryPlaceholder dead code · deleted

Deleted from Placeholders.swift (struct + #Preview + header comment). FXBarPlaceholder stays — it still marks the unshipped M15/M16/M17 FX slots.

B-11 · Auto BPM locks at 2× · addressed M11c.3 + PRD-BEATS

Octave hardened across M11c.3a–f (perceptual prior, reggae / hip-hop double-time rejection, genre-aware OctaveProfile) and PRD-BEATS rounds 5–10 (OctaveProfile::HipHop / DrumAndBass, integer-snap safety net). Real-music corpus now at dub-bpm/tests/fixtures/*.tsv (Classic = 11/94). Residual ambiguous tracks are handled by tap-to-grid; further automatic gains are gated on a learned beat tracker — see BPM-DETECTOR-V2-INVESTIGATION.md.

U-19 · Tap-to-grid · shipped M11c.3b / M11d.7

Deck-header BPM affordance: 1–2 taps relatch the downbeat (pure bar_phase rotation, BPM bit-identical); 3+ taps recompute tempo via analyze_beat_grid_from_taps. Persists user_tap rows and installs the grid in-place without full reanalysis. Binding spec is PRD-BEATS.md.

U-18 · Beat-grid overlay jitter · fixed (jitter killed end to end)

Grid + envelope now share one audio-clock-extrapolated playhead (EngineHostTimeMapping / PlayheadMarker) and render off the main thread (WaveformRenderThread). The os_signpost runbook in WAVEFORM-JITTER-CAPTURE.md stays wired for the next regression.

B-26 · Waveform scrub lag · closed M11d.5

Demand-driven Metal rendering (CVDisplayLink on the main run loop), input yield during press/drag, redundant peak-ingest skip when the playhead is stable, GPU catch-up when frames are deferred. Landed in 01291cd.

B-25 · BPM-grid polling never latches · closed M11d.5 round 4

Library-sourced beat grid is now the single source of truth; deck-load handshake delivers the final grid before the first Metal frame after load. Removed the ±0.02 BPM cross-deck drift and the redundant 100–400 ms per-load BPM analysis as a side effect.

B-10 · Key column tap-to-toggle · stale

Stale after browser column reset; current performance browser doesn't render Key by default. Re-open when Key returns via customizable columns; original fix (toggle in column header, not row cells) still stands.

U-13 · LibraryView header duplication · stale

Stale after list rewrite; the right-pane header count is gone. No action unless reintroduced.

C-26 · BPM lookup differs DeckHeader / LibraryView · closed M11d.5 round 4

Engine now adopts the library's track_beatgrids(is_active = 1) via a new LibraryBeatGrid parameter on DubEngine.loadTrack instead of re-running analyze_beat_grid. Both reads converge on the same SQLite row by construction.

C-29 · Table selection double-fires · obsolete

Obsolete after SwiftUI Table removal. Selection is owned by the row model and LibraryArrowKeyView now. Re-open only if a future NSTableView/SwiftUI Table paging rewrite reintroduces the binding echo.
